# Описание задания

## Модели

### 1. Chat

- id: int
- title: string not null
- created_at: time.Time

### 2. Message

- id: int
- chat_id: int
- text: string
- created_at: time.Time

Связь: один ко многим (один чат может содержать много сообщений)

## API

### 1. Создание чата

- URL: /chat/
- Метод: POST
- Тело:
  - title: string (обязательно, от 1 до 200 символов)
- Ответ:
  - Созданный чат

### 2. Отправка сообщения в чат

- URL: /chat/{chat_id}/messages/
- Метод: POST
- Тело:
  - text: string (обязательно, от 1 до 5000 символов)
- Ответ:
  - Созданное сообщение

### 3. Получение чата и последних N сообщений

- URL: /chat/{chat_id}/
- Метод: GET
- Параметры:
  - limit: int (по умолчанию 20, максимум 100)
- Ответ:
  - Чат
  - Последние N сообщений (отсортированы по created_at)

### 4. Удаление чата вместе со всеми сообщениями

- URL: /chat/{chat_id}/
- Метод: DELETE
- Ответ: 204 No Content (или json-статус)

При удалении чата должны удаляться все сообщения каскадно (на уровне БД/ORM).

## Технические требования

- Использовать `net/http`
- Работа с БД через `gorm`
- Использовать PostgreSQL
- Использовать миграции c помощью `goose`.
- Обернуть приложение в Docker и запустить через Docker Compose

Дополнительно:

- Логгирование
- Тесты (`httptest` + `testify`)
